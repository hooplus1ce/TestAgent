# 截图操作守则

## 核心规定

1. **截图只落盘，不送模型** — 所有对页面进行截图的操作（`tab.screenshot()`、`browser` 截图等），结果仅保存到本地项目路径，不得通过 `inspect_image` 或任何其他工具将截图数据发送给大模型。

2. **除非用户明确要求识别图片** — 仅当用户明确说出「识别这张图」「看下这个截图」「图片里有什么」等指令时，才调用图像分析工具（`inspect_image` 等）处理截图。

3. **默认行为** — 截图的目的是留档、调试、后续对比，而非让模型「看」页面。如需了解页面内容，优先使用 `tab.observe()`、`tab.ariaSnapshot()`、`dom_tree()` 等结构化 DOM 提取手段。

## 理由

当前运行环境的大模型不支持多模态输入。将截图发送给模型不仅浪费 token，而且模型无法理解图片内容，造成虚假输出。
