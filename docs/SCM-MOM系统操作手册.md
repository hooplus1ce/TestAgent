## 第 1 章 系统概述

### 1.1 系统定位

本系统是诺贝科技（中山）有限公司的 **SCM（供应链）/MOM（制造运营管理）/ERP 一体化平台**，采用 **按订单生产（Make-to-Order）** 模式，以销售订单为需求源头，通过 BOM 展开驱动采购与生产，覆盖"接单→排产→采购→生产→质检→出入库→出货→对账"全链路。

### 1.2 技术架构

| 项 | 说明 |
|---|---|
| 前端框架 | Ant Design Pro + React SPA |
| 表格渲染 | VTable Canvas（`.vtable` 元素，支持按值/按条件筛选） |
| 模块嵌入 | 业务页面以 iframe 内嵌旧版 spo 模块（`/scm-static/nuobei/next/spo/`） |
| 入口 | `HL_URL` 环境变量配置 |
| 设备集成 | AGV、立库、CTU、海康等智能仓储设备 |

### 1.3 模块全景

系统共 18 个一级模块、100+ 子功能，按业务类别归为五大类：

| 类别 | 模块 |
|---|---|
| 驱动入口 | 工作台、数据看板 |
| 主业务链 | 销售管理、采购管理、生产管理、托工管理 |
| 质量与仓储 | 质检管理、出入库管理、仓储管理、自动化仓储 |
| 闭环与变更 | 售后管理、变更管理、财务协同 |
| 基础支撑 | 基础配置、模具管理、标签管理、报表管理、统计分析、权限管理 |

## 第 2 章 快速入门

### 2.1 登录

1. 访问 SCM Admin 入口 URL。
2. 系统跳转至登录页（由 `HL_URL` 环境变量配置）。
3. 输入用户名、密码，填写图形验证码（纯数字）。
4. 点击「登录」。
5. 登录成功后自动跳转回 SCM Admin 主界面，页面标题显示"诺贝科技（中山）有限公司"。

> **会话过期**：系统检测到登录过期会弹出确认弹窗，需重新登录或刷新会话。

### 2.2 主界面布局

```
┌─────────────────────────────────────────────────┐
│  顶部导航（企业名称 / 用户）                        │
├──────────┬──────────────────────────────────────┤
│          │  顶部标签栏（多 Tab 切换，可关闭）        │
│ 左       ├──────────────────────────────────────┤
│ 侧       │                                      │
│ 菜       │     业务 iframe 区域                  │
│ 单       │   （各模块页面在此渲染）                │
│ 树       │                                      │
│          │                                      │
├──────────┴──────────────────────────────────────┤
│  系统状态栏                                       │
└─────────────────────────────────────────────────┘
```

- **左侧菜单树**：18 个一级模块，点击展开显示子功能。
- **顶部标签栏**：每进入一个模块开一个 Tab，可多 Tab 并行操作，点击 × 关闭。
- **业务 iframe**：模块页面在 iframe 内渲染，URL 形如 `.../spo/saleOrderCreate`。

### 2.3 菜单导航

- 点击一级菜单展开子菜单（再次点击折叠）。
- 点击子菜单项进入对应模块，顶部新增 Tab。
- 同一模块重复点击会聚焦已开 Tab 而非新开。

## 第 3 章 通用操作规范

### 3.1 新增页面通用布局

所有"新增"页面遵循统一结构：

- **顶部操作栏**：`返回` | `列表` | `新增` | `保存`（主按钮，蓝色） | 模块特定按钮（如`生成XX单`、`从商品库选择`）
- **基本信息区**：Ant Design 表单（DOM 原生输入框/下拉）
- **明细区**：VTable Canvas 表格，支持列头筛选/排序
- **全屏切换**：右上角全屏图标可放大编辑区

### 3.2 操作按钮说明

| 按钮 | 作用 |
|---|---|
| 返回 | 返回上一页 |
| 列表 | 跳转到对应列表页 |
| 新增 | 清空当前表单，开始新录单 |
| 保存 | 保存当前单据（主操作） |
| 生成XX单 | 由当前单据向下游生成单据（如销货单→生成出库订单） |
| 从商品库选择 | 弹窗选择商品加入明细 |
| 从销售订单选择 | 弹窗选择销售订单带入（生产需求单等） |

### 3.3 筛选区使用

列表页顶部为筛选区，**必须优先使用内联模式**（避免高级搜索弹窗）：

1. 若筛选区有模式切换按钮（`anticon-bars` 图标），点击切换到「内联模式」。
2. 内联模式下点击「展开▼」显示全部筛选字段。
3. 筛选字段为三段式：**字段名下拉 / 操作符下拉 / 值控件**。
   - 值控件为文本框：`free-text`，可自由输入。
   - 值控件为日期范围：`date-range`，用日期选择器。
   - 值控件为下拉：`must-select-option`，只能选已有选项，不能任意填写。
4. 设置筛选条件后点击「查询」，点击「重置」清空。

### 3.4 VTable 表格操作

VTable 是 canvas 渲染表格，无真实 DOM 节点：

- **排序**：点击列头排序图标（hover 显示）。
- **筛选**：点击列头筛选图标，弹出 `.vtable-filter-menu`，支持「按值筛选」/「按条件筛选」。
  - 按条件筛选：选择筛选类型（文本/数值/颜色等）+ 条件（等于/包含/介于等）+ 值。
- **列宽调整**：拖拽列头右边框。
- **滚动**：表格内容超出视口时滚动查看，列超出视口时水平滚动。
- **单元格交互**：按 `bodyBehavior` 分——复选框列点击勾选，链接/按钮列点击（或双击）触发动作。

### 3.5 弹窗交互

- **Ant Design 弹窗**（`.ant-modal`）：业务确认/详情，优先点「取消」或 × 关闭。
- **系统确认弹窗**（top 层 `.ant-confirm-body-wrapper`）：登录过期等系统级提示。
- **通知/消息**（`.ant-notification-notice` / `.ant-message-notice`）：操作反馈，自动消失或点 × 关闭。

> **重要**：与弹窗交互前，先用 `dom_tree` 获取弹窗完整结构，再精确点击按钮，避免 React 合成事件兼容性问题。

## 第 4 章 销售管理

销售管理是订单到收款（Order-to-Cash）流程的起点，包含 15 个子功能。

### 4.1 询价单管理

**路径**：销售管理 → 询价单新增 / 询价单

**功能**：记录客户询价，作为销售核价与报价的依据。

**操作**：
1. 进入「询价单新增」。
2. 填写客户信息、询价商品明细、期望价格。
3. 点击「保存」生成询价单。
4. 在「询价单」列表查询、跟进、转核价。

### 4.2 销售核价

**路径**：销售管理 → 销售核价

**功能**：对询价单进行核价，确定销售价格，核价通过后可转销售订单。

**操作**：
1. 打开销售核价列表，选择待核价单据。
2. 录入核价价格、有效期。
3. 提交核价，核价通过后报价生效。

### 4.3 打样单

**路径**：销售管理 → 打样单

**功能**：管理客户打样需求，跟踪样品制作与交付。

### 4.4 销售订单新增（核心）

**路径**：销售管理 → 销售订单新增

**功能**：录入客户正式销售订单，是后续生产、采购、出货所有环节的需求源头。

**页面字段**（基本信息）：

| 字段 | 说明 |
|---|---|
| 客户名称 | 关联客户主数据，下拉选择 |
| 客户编码 | 选择客户后自动带出 |
| 订单币别 | 人民币/外币，影响汇率换算 |
| 销售类型 | 普通销售等 |
| 交付日期 | 客户要求交付日期，驱动排产倒推 |
| 托运责任 | 客户提货/我方托运等，决定运费承担 |
| 交付地点 | 收货地点 |
| 计税类别 | 影响开票税率 |
| 客户订单号 | 客户方的订单号 |
| 包装要求 | 包装规范说明 |
| 外箱要求 | 外箱规范 |
| 出货条款 | FOB/CIF 等 |
| 发票单号 | 关联发票 |
| 跟单员 | 订单跟进负责人 |
| 业务员 | 销售归属人 |
| 收货联系人 | 收货方联系人 |
| 收货联系电话 | 收货方电话 |
| 备注 | 其他说明 |

**商品明细**：VTable 表格，通过「选择商品」添加行，录入数量、单价等。

**操作步骤**：
1. 点击「销售订单新增」进入。
2. 选择客户（客户编码自动带出）。
3. 填写订单条款（币别、销售类型、交付日期、托运责任、交付地点、计税类别）。
4. 录入订单标识（客户订单号、发票单号）。
5. 填写包装出货条款、跟单员、业务员、收货信息。
6. 在明细区添加商品，录入数量、单价。
7. 点击「保存」生成销售订单。

**按钮**：返回 / 列表 / 新增 / **保存**

### 4.5 销售订单查询与跟踪

| 报表 | 路径 | 用途 |
|---|---|---|
| 销售明细表 | 销售管理 → 销售明细表 | 查询销售订单明细行 |
| 未清销售订单 | 销售管理 → 未清销售订单 | 跟踪尚未完结的订单 |
| 受订未交反应表 | 销售管理 → 受订未交反应表 | 已接单未交付情况分析 |
| 销售统计表 | 销售管理 → 销售统计表 | 销售汇总统计 |
| 销货统计表 | 销售管理 → 销货统计表 | 实际销货汇总 |

### 4.6 出货计划

**路径**：销售管理 → 出货计划新增 / 出货计划

**功能**：根据销售订单制定出货计划，安排出货节奏，作为销货单的来源。

**操作**：
1. 「出货计划新增」选择销售订单及明细。
2. 设定计划出货日期、数量。
3. 保存生成出货计划。
4. 「出货计划」列表查询、调整、下发。

### 4.7 销货单（核心）

**路径**：销售管理 → 销货单新增 / 销货单

**功能**：执行销售出货，关联销售订单、出货计划、质检单，生成出库订单。

**页面字段**：

| 字段 | 说明 |
|---|---|
| 需求来源 | 销售订单/出货计划等 |
| 客户名称/编码 | 收货客户 |
| 托运责任 | 运费承担方 |
| 订单状态 | 单据流转状态 |
| 是否代发 | 是否代客户发货 |
| 运营方式 | 出货运营方式 |
| 质检 / 质检单号 | 是否质检及关联质检单 |
| 需要服务 | 是否需要附加服务 |
| 出仓 | 出仓方式 |
| 陆运服务 | 陆运安排 |
| 计划出库日期 | 计划出货时间 |
| 出库仓库 | 发货仓库 |
| 仓库服务商 | 仓库服务方 |
| 备注 / 销货单备注 | 说明 |

**明细列**：本次出库数量、关联质检单号、销售单号、出货计划单号、出库仓库、商品识别码、商品名称、品牌、型号、单位、计划出货数量、已出货数量、待出货数量、批次号、生产日期、单价、金额、币别、汇率、换算人民币金额等。

**操作步骤**：
1. 进入「销货单新增」。
2. 选择需求来源（销售订单/出货计划），带入明细。
3. 确认出库仓库、计划出库日期、托运责任。
4. 录入本次出库数量（不能超过待出货数量）。
5. 如需质检，关联质检单号。
6. 点击「生成出库订单」触发出库流程，或「保存」暂存。
7. 在「销货单」列表跟踪状态。

**按钮**：返回 / 列表 / **生成出库订单** / 添加出库订单 / 按明细选择货物 / 批量删除

### 4.8 出货清单

**路径**：销售管理 → 出货清单

**功能**：汇总出货明细，用于物流交接核对。

## 第 5 章 生产管理

生产管理是 MOM 核心，覆盖生产需求、计划、排产、制令、领料、缴库全流程，共 22 个子功能。

### 5.1 生产需求单新增（核心）

**路径**：生产管理 → 生产需求单新增

**功能**：由销售订单转化为生产需求，是生产计划的输入。

**页面字段**：

| 字段 | 说明 |
|---|---|
| 订单优先级 | 生产优先级 |
| 备注 | 需求说明 |

**操作步骤**：
1. 进入「生产需求单新增」。
2. 点击「从销售订单选择」，弹窗选择销售订单及明细带入。
3. （也可「从商品库选择」直接添加商品）
4. 设定订单优先级。
5. 点击「生成生产需求订单」提交。

**按钮**：返回 / 列表 / **生成生产需求订单** / 从商品库选择 / **从销售订单选择**

> **关键**：「从销售订单选择」按钮证实生产由销售订单驱动。

### 5.2 生产计划与制造排产

**路径**：生产管理 → 生产计划 / 制造排产 / 制造排产大屏

**功能**：
- **生产计划**：根据生产需求单编制生产计划，平衡产能。
- **制造排产**：将生产计划排到具体产线、时段。
- **制造排产大屏**：可视化展示排产甘特图。
- **车间进度跟踪**：实时跟踪车间生产进度。
- **上工记录**：记录工人上工情况，用于工资计算。

**操作**：
1. 「生产计划」中选择生产需求单，编制计划数量与时间。
2. 「制造排产」中将计划排到产线（关联制造部门、生产线）。
3. 通过大屏/车间进度跟踪执行情况。

### 5.3 制令单新增（核心）

**路径**：生产管理 → 制令单新增

**功能**：生成生产制造指令单（工单），是生产执行的依据，由销售订单/生产需求驱动，BOM 展开算物料。

**页面字段**：

| 字段 | 说明 |
|---|---|
| 制令类型 | 生产制令等 |
| 客户名称 / 客户编码 | 关联客户 |
| 成品编码 | 要生产的成品 |
| 商品型号 | 成品型号 |
| 数量 | 生产数量 |
| 来源单号 | 上游单据（生产需求单等） |
| 销售单号 | 关联销售订单 |
| 制造部门类型 | 生产执行部门 |
| 排产开始日期 | 计划开工 |
| 排产结束日期 | 计划完工 |
| 开工日期 | 实际开工 |
| 已缴库数量 | 生产完成缴库累计 |
| 待领料 | 待领物料 |
| 备注 | 说明 |

**操作步骤**：
1. 进入「制令单新增」。
2. 选择制令类型、客户、成品编码，录入数量。
3. 填写来源单号/销售单号（关联上游）。
4. 选择制造部门类型。
5. 点击「按BOM表计算」自动展开物料需求。
6. 设定排产开始/结束日期、开工日期。
7. 点击「保存」生成制令单。

**按钮**：返回 / 列表 / 新增 / **保存** / 删除

> **关键**：「按BOM表计算」+「来源单号/销售单号」证实制令由销售驱动、BOM 算物料。

### 5.4 生产备料与领料

**路径**：生产管理 → 生产备料新增 / 生产领料 / 生产补料 / 生产退料

**功能**：
- **生产备料**：根据制令单 BOM 提前备料。
- **生产领料**：从仓库领料到产线。
- **生产补料**：领料不足时补充领料。
- **生产退料**：剩余料退回仓库。
- **料件预计领用表**：预测物料领用情况。

**操作**：
1. 「生产备料新增」选择制令单，自动带出 BOM 物料清单。

## 第 17 章 各角色操作流程

### 17.1 销售员

```
询价单新增 → 销售核价 → 销售订单新增 → 跟单（未清销售订单/受订未交反应表）
  → 出货计划新增 → 销货单新增 → 应收对账单跟进
```

### 17.2 计划员

```
销售订单 → 生产需求单新增（从销售订单选择）→ 生产计划 → 制造排产
  → 料件预计领用表（缺料分析）→ 触发采购申请
```

### 17.3 采购员

```
物料申请单 → 采购订单新增 → 采购核价 → 采购缴库（进货）
  → 进货明细表/未清采购订单跟踪 → 应付对账单
```

### 17.4 生产主管

```
制令单新增（来源=销售单，按BOM计算）→ 生产备料 → 生产领料
  → 车间进度跟踪/上工记录 → 生产缴库新增（按制令单，缴库质检）
  → 生产动态表/工资明细
```

### 17.5 质检员

```
质检标准/项目维护 → 质检单新增（进货质检/缴库质检/工序质检）
  → 不良品统计表 → 质检结果统计表
```

### 17.6 仓管员

```
缴库订单（采购/托工/生产缴库入库）→ 出库单新增（销货出库）
  → 盘点单 → 调拨订单 → 库存调整/报损 → 库存信息/分仓报表
```

### 17.7 委外员

```
托工订单新增 → 托工领料 → 托工缴回 → 托工统计表
  （异常：托工补料/托工退料/托工退回）
```

### 17.8 财务

```
应收对账单（销售订单+出货订单汇总）→ 应付对账单（采购汇总）
  → 记账 → 成本审核/成本维护 → 进销存报表/总账凭证
```

## 第 18 章 附录

### 18.1 核心单据关联关系

```
询价单 → 销售核价 → 销售订单 ──┬→ 生产需求单 → 生产计划 → 制令单 → 生产领料 → 生产缴库
                              ├→（缺料）物料申请单 → 采购订单 → 采购缴库
                              └→ 出货计划 → 销货单 → 出库单 → 应收对账单
                                                                    ↓
                              采购缴库 → 应付对账单 ──────────→ 记账 → 总账
```

**关键字段关联**：
- 生产需求单.`从销售订单选择` ← 销售订单
- 制令单.`来源单号`/`销售单号` ← 销售订单/生产需求单
- 生产缴库.`按制令单添加` ← 制令单
- 采购订单明细.`受订单号` ← 销售订单
- 质检单.`关联单号` ← 采购进货/生产缴库/制令工序
- 销货单.`销售单号`/`出货计划单号`/`关联质检单号` ← 销售+出货计划+质检
- 应收对账单.`销售订单号`/`出货订单号` ← 销售+出货

### 18.2 单据状态说明

| 单据 | 典型状态字段 |
|---|---|
| 采购订单 | 订单状态 / 计划进货状态 / 进货状态 |
| 销货单 | 订单状态 |
| 制令单 | 已缴库数量 / 待领料 |
| 质检单 | 质检结果 |

### 18.3 通用操作快捷参考

| 操作 | 步骤 |
|---|---|
| 新增单据 | 模块新增页 → 填基本信息 → 添加明细 → 保存 |
| 查询单据 | 列表页 → 展开筛选区（内联模式）→ 设条件 → 查询 |

| 对账 | 财务协同 → 应收/应付对账单 |

### 18.4 注意事项

1. **筛选区必须用内联模式**：避免高级搜索弹窗，确保筛选字段暴露在 DOM 中。
2. **VTable 是 canvas**：无真实 DOM 节点，操作走坐标或工具封装，不能直接 querySelector。
3. **下拉框值控件**：`must-select-option` 类型只能选已有选项，不能任意填写。
4. **弹窗交互**：先 dom_tree 获取结构再精确点击，避免 React 事件兼容问题。
5. **会话过期**：弹出系统确认弹窗时，需重新登录或刷新会话。
6. **单据关联**：录入单据时尽量填写来源单号，保证全链路可追溯。
7. **BOM 变更**：BOM 修改必须走变更管理流程，不能直接改，确保追溯。
8. **质检前置**：采购进货、生产缴库前确认是否需质检，质检通过才能入库/出货。
