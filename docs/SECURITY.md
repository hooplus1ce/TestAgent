# drission-ui MCP 安全边界声明

⚠️ **重要提示：drission-ui MCP 不是安全边界**

本工具提供的以下措施仅为便利性质，**不构成安全保证**：

---

## 1. 不是安全边界的功能

### `DRISSION_UI_CAPS` 能力分组
- **目的**：减少 LLM 上下文 token 消耗，按需暴露工具
- **不是**：安全沙箱或权限控制机制
- **注意**：即使某个工具不在启用分组中，若 LLM 知道工具名称仍可能被调用（取决于 MCP 客户端实现）

### 环境变量中的密码
- **目的**：方便配置，避免硬编码
- **不是**：安全存储机制
- **注意**：环境变量可能被同一机器上的其他进程读取

---

## 2. 高风险操作

### `run_js` 工具（未来计划重命名为 `browser_run_js_unsafe`）
- **风险**：可执行任意 JavaScript，相当于 RCE（远程代码执行）
- **建议**：仅在完全信任的环境中使用
- **替代方案**：优先使用封装好的专用工具

### `click_xy` / `mouse_trail` 等坐标操作
- **风险**：可能点击到预期外的位置
- **建议**：优先使用基于选择器/文本的 `click` 工具

---

## 3. 建议的安全措施

### 开发环境
- 使用独立的测试账号
- 不要在生产数据上运行
- 考虑使用 `--headless` 模式避免误操作

### 生产环境（如需要）
- 隔离网络环境
- 使用专用浏览器配置文件
- 启用审计日志

---

## 4. 借鉴 Playwright MCP 的安全理念

> Playwright MCP is not a security boundary.
> — Playwright MCP 官方文档

我们认同这一理念：
- 明确告知用户什么是安全的、什么不是
- 不在文档中虚假宣传安全能力
- 高风险操作在命名中明确体现（如含 "unsafe"）
