# TestAgent 项目使用说明

本文档是项目级入口说明，覆盖环境准备、MCP 加载、浏览器连接、页面证据采集、测试用例生成、Excel 导出和常见问题。drission-ui MCP 的底层工具细节见 `mcp-servers/drission-ui/README.md`，更完整的历史操作手册见 `drission-ui-使用手册.md`。

## 1. 项目定位

TestAgent 是基于 MCP + DrissionPage 的企业系统测试用例生成工具集，目标是让 Claude、Codex 等 Agent 通过真实浏览器获取页面结构、表格数据、弹窗、按钮状态、网络响应和状态流转，再生成覆盖面更广、数据真实性更高的 WMS/MOM/ERP/SCM 功能测试用例。

项目分工：

| 层级 | 目录 | 作用 |
|------|------|------|
| MCP 工具层 | `mcp-servers/drission-ui/` | 浏览器连接、页面扫描、点击输入、表格、弹窗、网络监听、控制台、资源保存 |
| Claude 技能层 | `.claude/skills/test-case-generator-dp/` | 测试生成策略、字段规范、覆盖模型、Excel 导出脚本 |
| Codex 技能层 | `.agents/skills/test-case-generator-dp/` | Codex 适配层，复用 Claude 技能资产 |
| 项目级 MCP 配置 | `.mcp.json` | Claude Code 加载 `drission-ui` MCP |
| Codex MCP 配置 | `.codex/config.toml` | Codex 加载 `drission-ui` MCP |
| 测试用例数据 | `test_cases/` | 生成的用例 JSON、截图和 Markdown 结果 |
| 页面证据资源 | `resources/` | MCP 采集的页面模型、截图、网络记录等，本目录已 gitignore |

## 2. 环境准备

### 2.1 安装依赖

在项目根目录执行：

```bash
uv sync
```

项目依赖定义在 `pyproject.toml`，核心依赖包括：

- `DrissionPage`
- `mcp`
- `ddddocr`
- `httpx`
- `openpyxl`
- `pytest`

### 2.2 准备本机浏览器配置

仓库提交的是模板：

```text
configs/dp_configs.example.ini
```

首次使用时复制为本机配置：

```bash
cp configs/dp_configs.example.ini configs/dp_configs.ini
```

Windows PowerShell：

```powershell
Copy-Item configs/dp_configs.example.ini configs/dp_configs.ini
```

`configs/dp_configs.ini` 是本机配置文件，已加入 `.gitignore`，不要提交。默认 Chrome 用户数据目录是：

```text
dp_profile/
```

该目录也已 gitignore，适合保存本地登录态和浏览器缓存。

### 2.3 关键环境变量

大多数场景可直接使用默认值。需要切换环境或账号时使用环境变量覆盖：

| 变量 | 说明 |
|------|------|
| `HL_SCM_URL` | SCM Admin 首页 URL |
| `HL_SCM_BASE_URL` | SCM 站点根 URL，OCR 登录使用 |
| `HL_SCM_LOGIN_PAGE` | 登录页 Referer |
| `HL_SCM_USERNAME` | OCR 登录用户名 |
| `HL_SCM_USERPWD` | OCR 登录密码 |
| `HL_REMOTE_PORT` | Chrome 远程调试端口，默认 `9222` |
| `HL_TARGET_HINT` | 选择浏览器 tab 的标题提示 |
| `HL_SHOT_DIR` | MCP 证据资源保存目录，默认 `resources/` |
| `DRISSION_UI_CAPS` | MCP 工具分组，默认全量；可设为 `core,vtable,filter,observe,network` |

## 3. 启动与加载 MCP

### 3.1 MCP 入口

项目统一使用：

```text
drission_ui_mcp.py
```

Claude Code 的 `.mcp.json` 和 Codex 的 `.codex/config.toml` 都指向这个入口。

手动启动 MCP 服务时可执行：

```bash
uv run python drission_ui_mcp.py
```

通常不需要手动启动，Claude/Codex 会按项目 MCP 配置自动拉起。

### 3.2 Claude Code

项目根目录已有 `.mcp.json`。进入项目后检查 MCP 是否加载：

```bash
claude mcp list
```

应能看到：

```text
drission-ui
```

### 3.3 Codex

项目根目录已有 `.codex/config.toml`。在 Codex TUI 中可用：

```text
/mcp
```

或在 shell 中检查：

```bash
codex mcp list
```

应能看到 `drission-ui`。

## 4. 浏览器连接

推荐两种方式。

### 方式 A：让 DrissionPage 按配置接管或自启动

确保 `configs/dp_configs.ini` 存在，然后让 Agent 调用：

```text
connect()
check_session()
```

如果 `9222` 已有 Chrome 实例，则接管；没有实例时，DrissionPage 会按 `configs/dp_configs.ini` 自启动。

### 方式 B：手动启动 Chrome

Linux 示例：

```bash
google-chrome --remote-debugging-port=9222 --user-data-dir=dp_profile --start-maximized
```

Chromium 示例：

```bash
chromium --remote-debugging-port=9222 --user-data-dir=dp_profile --start-maximized
```

启动后让 Agent 调用：

```text
connect(port=9222)
check_session()
```

如果会话过期，调用：

```text
refresh_session()
```

## 5. 资源保存规则

MCP 采集的大文件不直接塞进对话上下文，优先使用 `filename` 参数保存到 `resources/`。

调用 `enter_module("模块名")` 后，简单文件名会自动保存到当前模块目录：

```text
enter_module("采购入库")
capture_page_model(filename="page-model.json")
```

保存路径：

```text
resources/采购入库/page-model.json
```

如果显式传子目录，则按指定路径保存：

```text
capture_page_model(filename="WMS/采购入库/page-model.json")
```

保存路径：

```text
resources/WMS/采购入库/page-model.json
```

常用证据文件建议：

| 文件 | 推荐内容 |
|------|----------|
| `page-model.json` | 页面整体模型，含动作、字段、弹窗、分页、表格 |
| `elements.json` | 可交互元素清单 |
| `table-scan.json` | 表格列、图标、交互能力 |
| `table-data.json` | 当前表格数据样本 |
| `network-record.json` | 操作过程的网络请求/响应 |
| `console.json` | 控制台错误和日志 |
| `screenshots/*.png` | 页面或关键弹窗截图 |

`resources/` 已 gitignore，不提交。

## 6. 标准测试用例生成流程

对 Agent 直接说：

```text
生成 <模块名> 的测试用例
```

例如：

```text
生成 领退料明细表 的测试用例
```

建议 Agent 按以下流程执行：

1. `connect()` 接管浏览器。
2. `check_session()` 检查登录态。
3. `enter_module("<模块名>", expand_filter=True)` 进入二级菜单模块。
4. `capture_page_model(filename="page-model.json")` 保存页面整体模型。
5. `scan_filter_fields()` 获取筛选字段、操作符和值域。
6. `scan_table(kind="auto", filename="table-scan.json")` 获取表格结构。
7. `get_all_table_data(kind="auto", filename="table-data.json")` 获取真实数据样本。
8. 对每个按钮、页签、行操作执行 `observe_start -> action -> observe_wait`。
9. 需要接口证据时使用 `network_record_start -> action -> network_record_stop` 或 `listen_start -> action -> listen_wait`。
10. 先建立覆盖模型，再生成正式用例 JSON。
11. 用 Excel 导出脚本生成最终文档。

重要原则：

- 正式用例只写基于真实页面证据验证过的场景。
- 未验证内容标记为 `[待确认]`。
- 不把 VTable、MCP 方法名、选择器等技术细节写入用户可读的测试步骤。
- 自动化建议字段可以说明使用哪些 MCP 工具做断言。

## 7. 常用 MCP 工具组合

### 7.1 页面资产采集

```text
capture_page_model(filename="page-model.json")
scan_page_elements(filename="elements.json")
scan_toolbar_actions()
scan_form_fields(scope="all")
scan_modal()
scan_drawer()
scan_pagination()
```

### 7.2 筛选区

```text
scan_filter_fields()
select_option(field_name="状态", option_text="已审核")
select_date_range(field_name="创建时间", start_date="2026/07/01", end_date="2026/07/08")
```

### 7.3 表格

```text
scan_table(kind="auto", filename="table-scan.json")
get_table_values(column_title="单号", kind="auto")
get_all_table_data(kind="auto", filename="table-data.json")
click_table_cell(row=0, column_title="单号", kind="auto")
scan_action_availability_by_selection(row=0, col=0)
```

### 7.4 动作探索

推荐使用聚合工具：

```text
explore_action(action="click", locator="text:新增", listen_targets="gateway")
```

等价目标是：

```text
observe_start(signals=["modal","notification","message","tab","url","network"], listen_targets="gateway")
click(...)
observe_wait(timeout=8)
capture_page_model(...)
```

### 7.5 网络和控制台

```text
network_record_start(targets="gateway", method="POST")
# 执行业务动作
network_record_stop(timeout=5, max_packets=20)
network_record_export(filename="network-record.json")

browser_console_messages(level="error", filename="console.json")
```

### 7.6 截图和 PDF

```text
screenshot()
screenshot(path="resources/模块名/screenshots/modal.png")
browser_save_pdf(filename="page.pdf")
```

生成的正式用例 JSON 放在：

```text

```

```text

test_cases/LTLMXB/LTLMXB_02_页签表格类.json
```

导出 Excel：

```bash
uv run python .claude/skills/test-case-generator-dp/scripts/generate_from_json.py test_cases/<MODULE_PINYIN>/*.json
```

```bash
uv run python .claude/skills/test-case-generator-dp/scripts/generate_from_json.py test_cases/LTLMXB/*.json
```

## 10. 常见问题

### MCP 没有加载

检查：

```bash
claude mcp list
codex mcp list
```

确认 `.mcp.json` 和 `.codex/config.toml` 中的命令都指向：

```text
uv run python drission_ui_mcp.py
```

### 浏览器连接失败

检查 `9222` 端口是否可用：

```bash
curl http://127.0.0.1:9222/json
```

如果没有响应，手动启动 Chrome，或确认 `configs/dp_configs.ini` 存在。

### 找不到业务 iframe

先确认已进入模块：

```text
enter_module("<模块名>", expand_filter=True)
get_active_frame()
```

如果仍失败，可能是菜单文本不匹配、页面未加载完成或登录态失效。

### 筛选字段不完整

使用：

```text
enter_module("<模块名>", expand_filter=True)
scan_filter_fields()
```

该工具会尽量切换为内联筛选区并展开字段。不要直接用 raw JS 临时抓筛选项。

### VTable 表格数据读不到

先扫描表格：

```text
scan_table(kind="auto")
```

VTable 是 canvas 渲染，完整行数据通常通过列值重建：

```text
get_table_values(column_title="单号", kind="auto")
get_all_table_data(kind="auto")
```

如果页面使用虚拟滚动或懒加载，采集结果只代表当前实例可读数据，需要结合分页、滚动或接口证据补充。

### 点击后漏掉 toast 或弹窗

不要点击后再观察。使用：

```text
observe_start(...)
click(...)
observe_wait(...)
```

或直接使用：

```text
explore_action(...)
```

### 会话过期

```text
check_session()
refresh_session()
```

`refresh_session()` 会重新 OCR 登录、清缓存、注入 cookie 并刷新页面。

## 11. 维护约定

- `configs/dp_configs.ini` 是本机配置，不提交。
- `dp_profile/` 是本机 Chrome 用户数据目录，不提交。
- `resources/` 是页面证据资源目录，不提交。
- `*.xlsx` 是导出结果，不提交。
- `.claude/skills/test-case-generator-dp/` 是 Claude 侧技能源。
- `.agents/skills/test-case-generator-dp/` 是 Codex 适配层，应尽量复用 `.claude` 中的规范和脚本。
- MCP 工具变更后要更新：
  - `mcp-servers/drission-ui/README.md`
  - `tests/test_server.py`
  - 必要时更新本说明文档

## 12. 推荐的一句话使用方式

打开项目、确认 MCP 已加载并连接浏览器后，对 Agent 说：

```text
使用 test-case-generator-dp，为 <二级菜单模块名> 生成测试用例。先采集页面模型和真实数据，建立覆盖模型，只生成已验证场景，最后导出 Excel。
```
