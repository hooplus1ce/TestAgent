# 探索式增量测试用例生成工作流程

> **新架构优势**: 数据与逻辑完全分离，边探索边积累，随时可导出

## 🔴 第一铁律：探索驱动，禁止臆造

```
任何未经浏览器实际点击、验证、观察过的功能，绝对不允许写进测试用例！

探索到什么 → 写什么
没探索到 → 绝对不写

生产缴库明细实例：
✅ 已验证存在：筛选功能、导出/打印按钮、列宽拖拽、横向滚动
❌ 实际不存在：分页功能、排序功能
→ 测试用例中绝对不能出现分页/排序相关场景！
```

| 探索步骤 | 要求 |
|---------|------|
| 🔍 点击按钮 | 真实点击，观察反应（弹窗/提示/请求） |
| 📊 操作表格 | 真实拖拽、滚动、点击，观察实际行为 |
| ⌨️ 输入字段 | 真实输入有效值、边界值、异常值 |
| 📝 写入用例 | 只写亲眼观察到的行为，不写任何猜测 |

## 🏗️ 新架构 vs 旧架构

| **对比项** | **旧架构** | **新架构 ✅** |
|-----------|-----------|-------------|
| 数据存储 | Python硬编码在脚本里 | 独立JSON文件（纯数据） |

| 人工编辑 | 修改Python易出错 | ✅ JSON易读易编辑 |

## 📝 JSON 用例编写规范（简化版）

### 标准字段说明

| **字段** | **类型** | **说明** | **示例** |
|---------|---------|---------|---------|
| `case_id` | string | 分类字母 + 3位序号 | `"F001"` |
| `case_title` | string | 动宾结构标题 | `"按缴入仓库筛选缴库记录"` |
| `priority` | string | 高级/中级/低级 | `"低级"` |
| `verify_point` | string | 一句话验证目标 | `"筛选后表格仅显示指定仓库的数据"` |
| `test_type` | string | 见8大类型 | `"功能测试"` / `"边界值测试"` |
| `function` | string | 所属功能点 | `"查询"` / `"打印"` / `"排序"` |
| `preconditions` | array | 前置条件列表 | `["已登录系统", "页面已加载"]` |
| `test_steps` | array | 步骤列表 | `["选择样品仓", "点击查询"]` |
| `test_data` | object | 键值对测试数据 | `{"缴入仓库": "样品仓"}` |
| `expected_result` | string | **可验证**的预期结果 | `"表格仅显示样品仓记录"` |
| `automation_suggestion` | string | drission-ui MCP 操作建议 | `"click_xy 点击查询按钮"` |

---

### 8大测试类型（test_type 取值）

```
1. 功能测试          - 正常功能验证
2. 边界值测试        - 零值/空值/最大值/最小值
3. 异常测试          - 非法输入/状态互斥/重复操作
4. 数据一致性测试    - 筛选+排序/分页后状态保持
5. 幂等性测试        - 防抖/重复提交/幂等验证
6. UI/交互测试       - 提示信息/样式/焦点管理
7. 性能测试          - 加载时间/响应时间
8. 业务流程测试      - 端到端完整流程
```

### ✨ 测试数据与预期结果格式规范

> **核心学习成果**：从企业标准用例中提炼的专业表述风格

| **字段** | JSON写法规范 | 效果说明 |
|---------|------------|---------|
| **测试数据** | 纯JSON对象<br>`{"缴入仓库": "样品仓", "制单时间": "2026-07-01"}` | ✅ 首行统一加「**表单数据**」<br>✅ 键值对用「**中文冒号**」分隔<br>✅ 多条数据自动换行<br>```<br>表单数据<br>缴入仓库：样品仓<br>制单时间：2026-07-01<br>``` |
| **预期结果** | 用 `\n` 分隔的编号列表字符串<br><br>```json<br>"expected_result": "1. 表格仅显示指定仓库的记录\n2. 所有行缴入仓库列值均为样品仓\n3. 无不符合条件的记录"<br>``` | ✅ 每条断言独立编号<br>✅ 表述明确可验证<br>✅ 拒绝「系统正常运行」类空话<br>✅ 1/2/3编号列表格式 |
