# 测试用例生成规范约定 (企业级管理系统)

> **版本**: v1.1
> **生效日期**: 2026-07-06
> **适用范围**: ERP/SCM/MOM/WMS 等企业级管理系统功能测试用例生成
> **目的**: 统一测试用例设计标准，确保测试覆盖率，提升用例质量

---

## 🔴 铁律：探索驱动，禁止臆造

> **任何未经浏览器实际探索验证的功能，绝对不允许写进测试用例！**

| ✅ 可以写 | ❌ 绝对不能写 |
|----------|-------------|
| 已经点击验证过的按钮 | 猜测的按钮功能 |
| 实际看到的筛选字段 | 假设的筛选逻辑 |
| 真实存在的表格操作（如列宽拖拽） | 臆造的功能（如分页/排序，实际不存在） |
| 触发过的提示/弹窗 | 想象的异常场景 |

**探索原则**：
1. 先动手点击验证功能存在
2. 观察实际行为和效果
3. 再对应设计测试用例
4. 没探索过 = 不写

## 一、核心设计原则

### 1.1 正向 + 反向结合原则
✅ **必须同时覆盖**：
- 正向测试（正常输入 → 预期结果）
- 反向测试（异常输入 → 正确拦截/报错）
- **比例要求**: 正向 : 反向 ≈ 2 : 1

### 1.2 分层覆盖原则
✅ **必须按以下层级设计**：
```
├── 字段级测试 (每个输入字段独立验证)
│   ├── 正常输入
│   ├── 边界值
│   ├── 异常值
│   └── 格式校验
├── 功能级测试 (单个按钮/操作)
│   ├── 正向功能
│   ├── 异常交互
│   └── 幂等性/防抖
├── 模块级测试 (组合操作)
│   ├── 多条件组合
│   ├── 状态流转
│   └── 数据一致性
└── 端到端测试 (完整业务流程)
    └── 完整业务链路验证
```

### 1.3 数据驱动原则
✅ **所有测试数据必须**：
- 来自真实页面探索，不是凭空臆造
- 覆盖业务场景中可能出现的真实数据
- 包含边界值、特殊值、异常值

| **子类** | **必须覆盖场景** | **示例** |
|---------|------------------|---------|
| **筛选查询** | 每个筛选字段独立验证 | 按仓库筛选、按日期筛选、按状态筛选 |
| | 多条件 AND 组合 | 仓库 + 状态、日期 + 数量 |
| | 多字段叠加筛选 | 3个及以上字段同时筛选 |
| | 筛选字段互斥 | 同一字段选多个互斥值 |
| | 重置/清空功能 | 点击重置后所有字段清空 |
| **按钮操作** | 每个按钮正向功能 | 导出、打印、新增、修改、删除 |
| | 前置条件不满足 | 未勾选行点击打印、无权限点击 |
| | 按钮状态切换 | 禁用/启用、选中行后按钮变化 |
| **表格操作** | 勾选/全选/反选 | 单选、全选、取消全选（**需实际验证，不存在不写**） |
| | 列宽调整 | 拖拽、边界值、撤销（**需实际验证，不存在不写**） |
| | 排序功能 | 升序、降序、多列排序（**需实际验证，不存在不写**） |
| | 滚动操作 | 横向滚动、纵向滚动、滚动后状态保持（**需实际验证，不存在不写**） |
| **弹窗操作** | 弹窗打开/关闭 | 打开、X关闭、取消按钮关闭 |
| | 弹窗内表单验证 | 必填校验、格式校验 |
| | 弹窗提交/取消 | 确定提交、取消关闭 |

### 2.2 边界值测试 (Boundary Value Test)
> **占比要求**: ≈ 25%
> **核心原则**: 每个输入字段都要测试边界

| **边界类型** | **必须覆盖场景** | **示例** |
|-------------|------------------|---------|
| **数值边界** | 最小值 | 数量 = 0、重量 = 0 |
| | 正常值范围 | 数量 = 100（在有效范围内） |
| | 最大值 | 数量 = 数据库字段最大值 |
| | 略超最大值 | 数量 = 最大值 + 1（应被拦截） |
| | 负值 | 数量 = -1、重量 = -100（应被拦截） |
| **字符串边界** | 空字符串 | 输入框留空 |
| | 最小长度 | 1个字符 |
| | 最大长度 | 数据库字段长度限制（如varchar(256)） |
| | 超长字符串 | 257个字符（应被截断或拦截） |
| | 前后空格 | 前后有空格的输入 |
| **日期边界** | 最小日期 | 0001-01-01 |
| | 最大日期 | 9999-12-31 |
| | 非法日期 | 2026-02-30、2026-13-01 |
| | 日期格式错误 | 2026/07/06（系统要求yyyy-mm-dd） |
| | 日期范围非法 | 开始日期 > 结束日期 |
| **特殊字符边界** | 空值 | NULL、空字符串 |
| | 空白字符 | 空格、制表符、换行符 |
| | 特殊符号 | !@#$%^&*()_+ |
| | SQL注入字符 | ' OR 1=1 -- |
| | XSS注入字符 | &lt;script&gt;alert(1)&lt;/script&gt; |
| | HTML特殊字符 | &lt; &gt; &amp; &quot; &apos; |
| | 中文/日文/韩文等非ASCII字符 | 测试、日本語、한국어 |
| | Emoji表情 | 😀🎉🔥 |

### 2.3 异常测试 (Exception Test)
> **占比要求**: ≈ 15%

| **异常类型** | **必须覆盖场景** | **示例** |
|-------------|------------------|---------|
| **非法输入** | 格式错误 | 数字字段输入中文、日期输入文本 |
| | 类型不匹配 | 整型字段输入小数 |
| | 长度超限 | 256字符字段输入300字符 |
| **状态异常** | 状态互斥 | 已审批单据再次审批、已删除单据修改 |
| | 状态流转非法 | 待审批 → 已删除（跳过中间状态） |
| **操作异常** | 重复操作 | 连续快速点击提交按钮 |
| | 并发操作 | 多用户同时修改同一条记录 |
| | 中途取消 | 提交后立即取消、弹窗加载中关闭 |
| **权限异常** | 越权访问 | 普通用户访问管理员功能 |
| | 无权限操作 | 按钮置灰但通过URL直接调用 |
| **数据异常** | 关联数据不存在 | 引用已删除的外键数据 |
| | 数据完整性破坏 | 必填字段为空 |
| **网络异常** | 网络超时 | 提交后网络断开 |
| | 网络中断后重试 | 断网重连后再次提交 |

---

### 2.4 数据一致性测试 (Data Consistency Test)
> **占比要求**: ≈ 10%

| **一致性类型** | **必须覆盖场景** | **说明** |
|---------------|------------------|---------|
| **筛选 + 排序** | 筛选后排序数据范围不变 | **需实际验证排序存在，不存在不写** |
| **筛选 + 全选** | 筛选后全选仅选中筛选结果 | **需实际验证复选框功能，不存在不写** |
| **多操作组合** | 筛选 → 排序 → 导出 | 导出数据应与当前页面显示一致 |
| **刷新后状态** | 页面刷新后筛选条件保持 | F5刷新后筛选条件不丢失 |
| **跨页面返回** | 进入详情后返回列表 | 返回后筛选/排序状态保持 |

---

### 2.5 幂等性与防抖测试 (Idempotency & Debounce Test)
> **占比要求**: ≈ 5%

| **测试类型** | **必须覆盖场景** | **示例** |
|-------------|------------------|---------|
| **防抖测试** | 快速连续点击按钮 | 1秒内连续点击打印5次，只执行1次 |
| | 按钮加载状态 | 点击后按钮置灰，请求完成后恢复 |
| | 防止重复提交 | 提交过程中再次点击被拦截 |
| **幂等性测试** | 重复提交结果一致 | 同一请求提交多次，最终状态相同 |
| | 重复操作无副作用 | 重复点击导出不会生成多份重复文件 |
| | 失败重试结果一致 | 第一次失败后重试，结果与成功时相同 |

---

### 2.6 UI/交互测试 (UI/Interaction Test)
> **占比要求**: ≈ 5%

| **测试类型** | **必须覆盖场景** | **示例** |
|-------------|------------------|---------|
| **提示信息** | 操作成功提示 | 操作后弹出绿色提示，3秒后自动消失 |
| | 操作失败提示 | 失败时弹出红色错误提示，显示具体原因 |
| | 校验提示 | 输入框下方实时红色校验提示 |
| **样式验证** | 按钮状态样式 | 禁用样式、悬停样式、点击样式 |
| | 表格样式 | 选中行高亮、斑马纹、列头样式 |
| | 响应式布局 | 窗口缩小/放大后布局不混乱 |
| **窗口操作** | 窗口最大化/最小化 | 系统窗口操作后页面渲染正常 |
| | 窗口缩放 | 浏览器缩放100%/125%/150%显示正常 |
| **焦点管理** | Tab键遍历 | Tab键按正确顺序遍历所有可交互元素 |
| | Enter键提交 | 焦点在输入框时Enter提交表单 |
| | ESC键关闭 | ESC键关闭弹窗/下拉菜单 |

---

### 2.7 性能测试 (Performance Test)
> **占比要求**: ≈ 5%

| **测试类型** | **必须覆盖场景** | **验收标准** |
|-------------|------------------|-------------|
| **页面加载** | 首次进入页面加载时间 | ≤ 2秒 |
| | 筛选后表格刷新时间 | ≤ 1秒 |
| | 大数据量首次渲染 | 500条数据 ≤ 3秒 |
| **操作响应** | 按钮点击响应时间 | ≤ 500ms |
| | 弹窗打开时间 | ≤ 300ms |
| | 下拉菜单展开时间 | ≤ 200ms |
| **滚动性能** | 大数据量滚动流畅度 | 60fps，无明显卡顿 |

---

### 2.8 业务流程端到端测试 (E2E Business Flow Test)
> **占比要求**: ≈ 5%

| **流程类型** | **必须覆盖场景** |
|-------------|------------------|
| **完整查询流程** | 进入页面 → 设置筛选条件 → 查询 → 验证结果 → 重置 |
| **完整操作流程** | 筛选 → 勾选 → 打印 → 导出 → 验证结果 |
| **状态流转流程** | 新建 → 提交 → 审批 → 修改 → 作废 完整链路 |
| **异常流程** | 提交 → 驳回 → 修改 → 重新提交 → 通过 |

## 三、用例编写规范

### 3.1 用例编号规则
```
{企业前缀}_{模块拼音缩写}_{分组字母}{3位序号}
├── 分组字母:
│   ├── F = 筛选查询 (Filter)
│   ├── A = 新增/添加 (Add)
│   ├── M = 修改/编辑 (Modify)
│   ├── D = 删除/作废 (Delete)
│   ├── I = 交互类 (Interaction: 按钮/弹窗/表格)
│   ├── B = 批量操作/导出/打印 (Batch)
│   ├── P = 页面级/性能/UI (Page)
│   └── E = 端到端业务流程 (End-to-End)
```

### 3.2 用例标题规范
✅ **动宾结构，简洁明确**
- ✅ 正确: "按缴入仓库筛选缴库记录"
- ✅ 正确: "重量输入负数验证拦截"
- ❌ 错误: "测试仓库筛选"
- ❌ 错误: "仓库功能测试"

### 3.3 级别分配决策树
```
是否阻塞主流程（不通过则核心业务无法进行）？
├── 是 → 高级
└── 否
    ├── 是否核心业务功能（增/删/改/提交/审批）？
    │   ├── 是 → 中级
    │   └── 否
    │       ├── 是否常见交互（查询/排序/筛选/分页/详情查看）？
    │       │   ├── 是 → 低级
    │       │   └── 否
    │       │       ├── 是否状态流转关键节点？
    │       │       │   ├── 是 → 中级
    │       │       │   └── 否 → 低级
    │       │       └── 是否边界值/异常值？
    │           ├── 是 → 低级
    │           └── 否 → 低级
```

### 3.4 验证点规范
✅ **一句话概括，明确可验证**
- ✅ 正确: "筛选后表格仅显示指定仓库的数据"
- ✅ 正确: "输入负数后弹出校验提示，不允许提交"
- ❌ 错误: "验证筛选功能正常"
- ❌ 错误: "系统正常运行"

### 3.5 前置条件规范
✅ **只描述测试开始前的初始状态，不含任何操作**
- ✅ 正确: "1. 已登录系统\n2. 在生产缴库明细页面\n3. 数据表格已加载"
- ❌ 错误: "已进入详情页"（这是操作后的状态，应放步骤）

### 3.6 测试步骤规范
✅ **编号从1开始连续递增，每步一个具体动作**
- ✅ 正确: "1. 在缴入仓库下拉选择样品仓\n2. 点击查询按钮"
- ❌ 错误: "设置筛选条件后查询"（跳步）

### 3.7 预期结果规范 (核心质量门)
✅ **必须满足：第三方读完能独立判断通过/不通过**

| ✅ 好的预期结果 | ❌ 坏的预期结果 |
|----------------|----------------|
| 表格仅显示缴入仓库为「样品仓」的记录 | 系统正常响应 |
| 弹出红色提示「重量不能为负数」 | 数据正确显示 |
| 按钮置灰，连续点击只触发1次请求 | 操作成功 |
| 筛选后按日期排序，仍只有样品仓记录 | 功能正常 |
| 翻页后筛选条件仍为「样品仓」 | 页面跳转正确 |

✅ **可验证信号类型**:
1. 数据范围验证（所有制单号包含XXX）
2. 计数验证（共3条，均为冲压车间）
3. 视觉变化验证（弹窗消失/按钮置灰）
4. 状态保持验证（翻页后条件不变）
5. 校验拦截验证（弹出提示，未提交）
6. 接口响应验证（`explore_action(listen_targets=...)` 获取 response.body）

## 六、用例去重规则

### 6.1 合并条件
**相同前置条件 + 相同验证点** → 合并为一条，步骤用「或」连接

### 6.2 合并示例
| 合并前 (2条) | 合并后 (1条) |
|-------------|-------------|
| 查看弹窗内「物料编码」列<br>预期: 显示正确 | 查看弹窗内明细数据<br>步骤: 查看物料编码 / 物料名称 / 规格型号列<br>预期: 各列均显示对应字段值，无空白错位 |
| 查看弹窗内「物料名称」列<br>预期: 显示正确 | |

### 6.3 不合并的情况
- 验证点不同
- 级别不同
- 前置条件不同
- 测试类型不同

## 七、输出格式要求

### 7.1 Excel 结构
- **Sheet 1**: 测试用例（19列完整格式）
- **Sheet 2**: 测试数据配置 + 用例统计汇总 + 字段说明 + 按钮列表 + 表格列定义

### 7.2 排序规则
按视觉布局 + 测试类型排序：
1. 筛选查询类 (F) → 按字段在页面上的出现顺序
2. 新增/修改/删除类 (A/M/D)
3. 按钮交互类 (I) → 按按钮在工具栏的顺序
4. 表格操作类 (I) → 按操作复杂度
5. 批量操作类 (B)
6. 页面级/性能/UI类 (P)
7. 端到端业务流程类 (E)
