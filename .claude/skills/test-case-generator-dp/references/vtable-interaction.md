# VTable 交互探索规范

本文件只描述 AI 在生成测试用例时应如何使用 `drissionpage-mcp` MCP 的 VTable 能力，不记录 VTable 内部 JS、React fiber、坐标换算或 DOM 选择器。底层实现属于 MCP 工具层。

## 一、工具契约

AI 只依赖 MCP 返回的结构化结果：

| 能力 | 用途 | 要求 |
|---|---|---|
| `scan_table(kind="auto")` | 获取表格类型、列定义、可见数据、可交互列、列头图标 | 探索表格前必须先调用 |
| `click_table_cell(...)` | 点击复选框、链接、按钮、列头图标或可编辑单元格 | iframe、滚动、坐标换算由工具处理 |
| `get_table_values(...)` / `get_column_values(...)` | 读取用户可见数据 | 用于筛选、排序、编辑后的断言 |
| `observe_start` / `observe_wait` | 捕获点击后的弹窗、消息、Tab、URL、接口变化 | 每次点击前后必须使用 |
| 表格专项工具 | 列宽调整、横向滚动、单元格编辑、虚拟下拉识别 | 若工具不存在，记录能力缺口，不写 raw JS 兜底 |

## 二、探索顺序

1. 进入目标模块后先调用 `scan_table(kind="auto")`。
2. 根据返回列定义识别：复选框列、文本列、链接/按钮列、可编辑列、带排序/筛选图标的列。
3. 对每类可交互列至少探索一个真实样本行，动作前后都使用 `observe_start` / `observe_wait`。
4. 读取真实页面数据作为测试数据，禁止手造业务数据。
5. 将实际观察到的行为转化为用例，不为未出现的排序、分页、编辑器、批量操作补写场景。

## 三、常见交互策略

| 场景 | AI 层策略 | 生成用例关注点 |
|---|---|---|
| 行选择 | 使用 `click_table_cell` 点击复选框列 | 选中态变化、按钮启用/禁用、批量操作前置条件 |
| 链接/按钮单元格 | 使用 `click_table_cell` 触发，再观察反馈 | 详情弹窗、页面跳转、消息提示、接口响应 |
| 列头排序/筛选 | 使用 MCP 暴露的列头图标能力 | 数据顺序变化、筛选条件生效、空结果 |
| 横向滚动 | 使用表格专项工具或 `scan_table` 返回的滚动能力 | 右侧列可见性、固定列状态、滚动后数据一致 |
| 列宽调整 | 使用表格专项工具 | 宽度变化、内容展示、刷新后是否保持 |
| 单元格编辑/虚拟下拉 | 使用 MCP 返回的编辑器/浮层结构化结果 | 可选值来源、保存校验、编辑后表格值与接口结果 |

## 四、降级规则

当 `scan_table(kind="auto")` 返回失败或无法识别表格结构时：

1. 截图存档，并使用 `scan_page_elements()` / `dom_tree()` 获取可见标题和按钮。
2. 只生成低级展示类或页面级用例。
3. 跳过复选框、链接、排序、筛选、编辑等需要精确表格坐标的交互用例。
4. 在备注中标注「表格结构未被 MCP 识别，断言基于可见页面信息，建议人工复核」。

## 五、禁止项

- 禁止在 skill 中写浏览器内部脚本、DOM 查询、VTable 私有对象、React 内部对象或坐标换算。
- 禁止 AI 手动计算 iframe 偏移、canvas 坐标或滚动距离。
- 禁止为了生成更多用例而绕过真实 UI 行为直接修改前端状态。
- 禁止把 VTable 内部 API 当成测试步骤或预期结果写入 Excel。

若确实需要这些能力，先把能力沉淀到 `drissionpage-mcp` MCP 工具，再由 skill 通过稳定工具名调用。

## 六、DFS 衍生规则

按钮或单元格点击后的产物作为 DFS 子节点递归遍历：

| 类型 | 判断来源 | 处理方式 |
|---|---|---|
| 交互弹窗 | `observe_wait` 返回弹窗 | 扫描弹窗字段和按钮，关闭后回到原页面 |
| 业务确认弹窗 | `observe_wait` 返回确认类弹窗 | 分别覆盖取消、关闭、确定 |
| 消息提醒 | `observe_wait` 返回通知/消息 | 提取真实文案并写入预期结果 |
| 页面跳转或新 Tab | `observe_wait` 返回 Tab/URL 变化 | 重新扫描页面结构，探索后回收上下文 |
| 内容变更 | 未出现弹窗但页面结构变化 | 重新扫描 DOM/表格并继续探索 |

衍生用例的前置条件只写测试开始前的初始状态；完整操作路径必须写入测试步骤。
